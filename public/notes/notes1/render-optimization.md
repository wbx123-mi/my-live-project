# 渲染优化 (Renderer Optimization)

---

> 核心思想：编译指导运行 (Compiler-Informed Runtime)
> Vue 3 性能飞跃的根本原因在于：利用编译器在编译阶段收集的信息，精准指导运行时的更新动作。它通过以下三个核心机制实现了从“全量比对”到“精准打击”的跨越。
> 编译指导运行：利用编译阶段收集的信息，精准指导运行时的更新动作。

### 1. 静态提升 (Static Hoisting)

- **原理**：编译器识别出模板中永远不会变的节点（纯静态 HTML），将其提取到渲染函数之外。
- **面试话术**：“Vue 3 将静态节点从渲染函数中提升到全局变量中。在组件更新时，Vue 直接复用内存中现成的 VNode 引用，避免了每次渲染都要创建和销毁虚拟 DOM 对象，极大地减轻了内存压力。”
- **效果**：内存开销降至最低，Diff 算法遇到这类节点时直接跳过。

### 2. 补丁标志 (Patch Flags)

- **原理**：在编译动态节点时，编译器会根据变化的类型（如 class、id、text）生成一个位掩码 (Bitmask) 标记
- **面试话术**： “Vue 3 的 VNode 带有 patchFlag。渲染器在 Diff 时不再全量比对所有属性，而是根据标记位进行定向更新。例如，如果标记为 TEXT，它只比对文字；如果是 CLASS，它只比对类名。这种靶向更新让 Diff 性能不再受模板大小影响，只与动态内容的多少相关。

- **标记位对照表**：
  | 标志位 | 含义 | 说明 |
  | :--- | :--- | :--- |
  | 1 (TEXT) | 动态文本 | 只比对 textContent |
  | 2 (CLASS) | 动态 Class | 只比对 className |

### 3. 块结构 (Block Tree)

- **原理**：Vue 3 将模板按 v-if 或 v-for 切分为不同的“块 (Block)”。每个 Block 内部维护一个打平的数组 dynamicChildren
- **面试话术**：“Block Tree 将复杂的嵌套树结构在逻辑上打平。更新时，渲染器不再一层层递归扫描子节点，而是直接通过数组‘瞬移’到动态节点上进行处理。即使你的模板嵌套了 10 层深，只要里面只有 1 个动态节点，Vue 的更新也只需要 1 次操作。”
- **底层实现**：通过 `dynamicChildren` 数组实现“路径缩短”。

### 4. 总结

总结：如何向面试官收尾？
“总的来说，Vue 3 的优化是体系化的：

静态提升解决了‘存量’问题（不动的就不管）。

补丁标志解决了‘变量’质量问题（变哪里比哪里）。

块结构解决了‘路径’问题（跳过静态路径，直达目标）。 这套组合拳使得 Vue 3 在处理复杂大型项目时，渲染效率相比 Vue 2 有了质的提升。”
