# 🧩 插槽（Slots）底层实现：渲染函数与对象

> **一句话本质**：插槽在 Vue 底层不是简单的“内容搬运”，而是父组件向子组件传递了一个包含多个**渲染函数**的对象。

---

## 1. 插槽运行的三步曲

### ❶ 父组件：包装“函数包”

当你写 `<Child><p>Hello</p></Child>` 时，编译器并不会在父组件里立即生成 `p` 标签，而是将其包装成一个 **Slots 对象**，作为参数传递给子组件。

### ❷ 子组件:预设“出餐口”

子组件模板里的 <slot name="header"></slot> 在底层对应一次函数查找与调用。

### ❸ 最终阶段:调用与注入

子组件在执行自己的 render 函数时，会经历以下逻辑：

识别：扫描到需要渲染 header 插槽。

查找：从 $slots 对象里找到 header 这个 Key。

执行：运行对应的渲染函数 slots.header()。

嵌入：得到产生的 VNode（虚拟 DOM） 并嵌入到子组件的虚拟 DOM 树中。

```javascript
// 传递给子组件的 $slots 参数
const slots = {
  // 属性值是一个待执行的渲染函数 (返回 VNode)
  default: () => h('p', 'Hello'),
  header: () => h('h1', '标题'),
};
```

### ⚡️ 高频面试：插槽底层为什么传“函数”而非“VNode”？

| 比较维度         | 方案 A：直接传 VNode (快照)                                                                  | 方案 B：传渲染函数 (Vue 的选择)                                                    |
| :--------------- | :------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------- |
| **响应式实时性** | **数据滞后**。VNode 是在父组件渲染时生成的。如果父组件数据改变，子组件拿到的依然是旧节点。   | **实时获取**。子组件渲染时才调用函数，每次执行都会读取最新数据，确保状态完全同步。 |
| **性能开销**     | **无谓浪费**。无论子组件是否显示该插槽（如被 `v-if` 隐藏），插槽内容都会在父组件中提前运行。 | **延迟执行 (Lazy)**。只有子组件确定需要显示插槽时才会触发函数调用，节省计算资源。  |
| **数据流向**     | **单向静态**。父组件单向丢给子组件一个死的结果。                                             | **双向联动**。支持“作用域插槽”，子组件可以在调用函数时传参，实现数据倒灌。         |

---

#### 💡 深度原理解析：

1.  **关于响应式追踪**：如果传渲染函数，子组件在执行 `slot()` 时会触发父组件数据的 `getter`。这意味着子组件会将该数据收集为自己的依赖。当父组件数据变化时，子组件会自动重新触发渲染。
2.  **关于延迟执行**：这是一种 **Lazy Evaluation（惰性求值）** 策略。通过函数封装，Vue 将插槽内容的“运行权”交给了子组件，从而实现了更精细的渲染控制。
