# JavaScript 核心笔记：闭包 (Closure) 深度解析

## 1. 什么是闭包？

**官方定义：** 一个函数和对其周围状态（Lexical Environment，词法环境）的引用捆绑在一起，这样的组合就是闭包。

**通俗理解：** 闭包让你可以在一个内层函数中访问到其外层函数的作用域。在 JavaScript 中，每当创建一个函数，闭包就会在函数创建时刻被生成。

**深度视角：** 广义上，JS 中所有函数都是闭包，因为它们在创建时就绑定了词法环境。 狭义上（实战中），闭包特指那些在外部函数执行后，依然持有着外部变量引用的内部函数。这种机制让我们可以模拟私有变量，但也带来了内存管理上的挑战。

---

## 2. 闭包的三个核心特征

1. **函数嵌套**：外部函数中嵌套内部函数。
2. **内部引用**：内部函数引用了外部函数的变量。
3. **生命周期延长**：外部函数执行完毕后，由于内部函数仍在引用其变量，导致外部函数的作用域不会被垃圾回收机制（GC）销毁。

---

## 3. 代码示例与执行过程

```javascript
function createCounter() {
  let count = 0; // 这个变量被“封闭”在闭包里
  return function () {
    count++;
    console.log(count);
  };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
```
